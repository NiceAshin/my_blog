# YOLO 选型

> 预计阅读时间：5 分钟

## 1. 需求分析
在选择 YOLO 系列模型前，先明确业务目标：
- **检测目标种类**：类别数越多，需要越强的特征表达能力。
- **性能约束**：实时性（FPS）、延迟上限、可用算力（GPU/CPU/NPU）。
- **部署环境**：云端训练 + 边缘推理？是否需要移动端或嵌入式部署？
- **数据规模**：训练集大小、标注质量、类别不平衡等因素会影响模型收敛。

## 2. 版本对比
| 版本 | 特点 | 适用场景 |
| --- | --- | --- |
| **YOLOv3** | 结构相对简单，Darknet-53 主干；推理稳定，生态成熟。 | 老旧 GPU/CPU 环境；对精度要求适中且需快速落地的项目。 |
| **YOLOv4** | 引入 CSPDarknet、PANet、Mish 等改进，精度与速度平衡提升。 | 工业质检、视频监控等需要较高精度但仍追求实时性的场景。 |
| **YOLOv5** | PyTorch 实现，支持从 nano 到 x 的多种规模，训练/部署工具链完善。 | 需要快速迭代、AutoML、导出 ONNX/TensorRT/NCNN 的团队。 |
| **YOLOv6** | 面向工业场景优化，使用 EfficientRep + Rep-PAN，着重推理效率。 | ARM/边缘设备，要求高吞吐、低延迟的实时检测。 |
| **YOLOv7** | 对多标签、模型重参数化、任务适配（例如 pose）支持更好。 | 需要最新特性且硬件允许更复杂模型的项目。 |
| **YOLOv8** | Ultralytics 重构，原生支持检测/分割/姿态估计，多任务统一接口。 | 希望在单一框架内覆盖多任务，追求易用性与快速部署。 |

## 3. 选型建议流程
1. **先以轻量级模型做基线**：可选择 YOLOv5n/YOLOv8n 快速验证数据集质量和流程。
2. **精度与速度双向调优**：若精度不足，逐步换成更大 backbone；若速度不足，尝试剪枝、量化或选择 YOLOv6/YOLOv5n + TensorRT。
3. **关注部署工具链**：YOLOv5/YOLOv8 在导出 ONNX、TensorRT、OpenVINO、CoreML 上支持成熟；若目标是 MCU/FPGA，可评估 YOLOv4-tiny、YOLOv5n + 专用编译器。
4. **结合训练数据特性**：对小目标或极端场景，可增配多尺度训练、Mosaic 数据增强，并在模型上启用高分辨率输入。
5. **闭环评估**：在真实业务环境（视频流/嵌入式设备）做端到端测试，关注 mAP、召回率、FPS、资源占用等指标。

## 4. 实践小贴士
- **数据清洗优先**：高质量标注比盲目堆砌模型更有效。
- **多阶段部署**：先在云端试跑，再迁移到边缘设备，并通过 A/B 测试逐步替换。
- **运维监控**：上线后监控检测结果的置信度分布和失败样本，及时回流增量数据做再训练。
- **社区资源**：Ultralytics 官方仓库、OpenMMLab YOLOX、腾讯开源的 YOLOv6 都提供了训练脚本和预训练权重，可按需复用。
